# 1. Purpose

Define how the Office Partner (and humans) name, classify, and organize business files so any file can be found in seconds.

---

# 2. Naming Convention

`YYYY-MM-DD_Category_Description.ext`

| Rule | Example |
|------|---------|
| Date first, ISO format | 2026-07-14_Documents_Client_Contract.docx |
| Category matches the folder category | Documents, Spreadsheets, Presentations, Images, Archives, Data, Other |
| Description in words separated by underscores | Monthly_Tracker, Meeting_Minutes |
| Lowercase extensions | .docx not .DOCX |

Files that already follow the convention are never renamed again.

---

# 3. Category Folders

| Folder | Contents |
|--------|----------|
| Documents | doc, docx, pdf, txt, md, rtf, odt |
| Spreadsheets | xls, xlsx, csv, ods |
| Presentations | ppt, pptx, odp |
| Images | png, jpg, jpeg, gif, svg, bmp, webp |
| Archives | zip, rar, 7z, tar, gz |
| Data | json, xml, sqlite3, db |
| Other | anything else |

---

# 4. Safety Rules

1. Organizing never deletes files.
2. Duplicates are reported to a human, never auto-removed.
3. Moves stay inside the folder being organized.
4. Every organized folder gets an INDEX.md generated by the file organizer.
5. Official AOS records and registers are never reorganized by tools — humans only.
