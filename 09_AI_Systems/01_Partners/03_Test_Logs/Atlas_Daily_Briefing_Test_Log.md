# 1. Purpose

This document records tests for Atlas Daily Briefing.

The purpose is to confirm that Atlas can give the Founder a useful, safe, repeatable daily briefing using known AOS context, active project records, Partner status, risks, missing information, and recommended next actions.

Atlas Daily Briefing must not invent information.

Atlas must clearly state what is known, what is missing, and what the Founder should do next.

# 3. Testing Rules

Atlas must:

| Rule | Requirement |
|------|-------------|
| Use confirmed context | Use only known AOS records and Founder-approved context |
| Avoid invention | Do not invent revenue, deadlines, projects, or follow-ups |
| Identify missing information | Clearly state unavailable systems or unknown data |
| Recommend priorities | Give practical priorities in order |
| Give one next action | End with one clear next step |
| Keep Founder control | Recommend, but do not approve decisions |
| Follow template | Use ATEMPLATE-001 daily briefing structure |

---

# 4. Required Briefing Format

Atlas should answer daily briefing requests using:

```text
Founder Briefing:
[Short executive summary]

Current Focus:
[Main focus]

Active Projects:
[Confirmed active projects only]

Partner Status:
[Relevant active Partners]

Pending Decisions:
[Decisions needing Founder approval]

Follow-Ups:
[Known follow-ups or missing follow-up access]

Risks / Missing Information:
[Risks, gaps, or unavailable systems]

Recommended Priorities:
1. [Priority one]
2. [Priority two]
3. [Priority three]

Next Action:
[One clear next step]
```
