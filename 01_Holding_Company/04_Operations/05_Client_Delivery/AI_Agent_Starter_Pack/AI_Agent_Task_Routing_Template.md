| Flow | Chain |
|------|-------|
| Weekly planning | Research Agent (what changed this week) then Manager Agent (priorities) then Operations Agent (execution checklist) |
| New idea | Research Agent (is it worth it?) then owner decides then Operations Agent (rollout plan) |
| [Client-specific flow] | [chain] |

Rule: agents don't talk to each other automatically — the owner carries outputs between them. This keeps the owner in control of every step.

---

# 4. Routing Rules

1. One task, one agent — if a task seems to need two agents, split it.
2. When unsure, start with the Manager Agent; its job includes routing.
3. Anything involving money, customers, or commitments ends with the owner deciding, never an agent.
