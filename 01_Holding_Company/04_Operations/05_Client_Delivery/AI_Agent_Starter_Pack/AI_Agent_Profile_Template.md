# 1. How to Use

Fill one copy of this template per agent (3 per engagement). The profile is written from the client's intake answers and reviewed by the Founder before the prompt (AIAD-004) is written from it.

---

# 2. Agent Profile

| Field | Entry |
|-------|-------|
| Agent name | [e.g., "Maya — Manager Agent for (Client Business)"] |
| Agent type | [Manager / Research-Analyst / Operations-Implementation] |
| Client business | [Business name and one-line description] |
| Mission | [One sentence: what this agent exists to do for this business] |
| Daily responsibilities | [3-6 bullet points from intake answers] |
| Inputs it works from | [What the client gives it: notes, lists, questions, meeting summaries] |
| Outputs it produces | [What it returns: prioritized lists, summaries, checklists, reports] |
| Tone and language | [From intake Q21: language(s), formal/casual] |
| What it must always do | [e.g., ask clarifying questions when input is vague; end with clear next actions] |
| What it must never do | [From intake Q20 + AIAD-007 safety rules: never send, never spend, never contact anyone] |
| Escalation rule | [When it must stop and hand the decision back to the owner] |
| Example task 1 | [A real task from the client's business, with expected output shape] |
| Example task 2 | [A second real task] |

---

# 3. Quality Checklist Before Approving a Profile

1. Mission fits in one sentence a non-technical owner understands.
2. Every responsibility traces back to an intake answer — nothing invented.
3. The never-do list includes the AIAD-007 baseline rules.
4. Both example tasks are real tasks from this client, not generic ones.
