# 1. Purpose

The Project Register is the official list of projects inside ALSAKKAF HOLDING GROUP.

It tracks proposed, approved, active, paused, completed, cancelled, rejected, and archived projects.

The register helps AOS understand:

- Which projects exist
- What each project is trying to achieve
- Who owns each project
- What status each project has
- Which Partners support each project
- Which records are related to each project
- Whether the project is aligned with AOS

No official project should exist without a Project Register entry.

# 2. Project Status Definitions

| Status | Meaning |
|--------|---------|
| Idea | Not yet structured |
| Proposed | Project has been suggested |
| Researching | Information is being gathered |
| Designed | Project structure is documented |
| Approved | Project is approved to begin |
| Active | Project is currently being executed |
| Paused | Project is temporarily stopped |
| Completed | Project deliverables are finished |
| Archived | Project is closed and stored |
| Rejected | Project was reviewed and not approved |
| Cancelled | Project began but was stopped |

# 4. Project Register Rules

1. Every official project must have a Project ID.
2. Project IDs must not be reused.
3. Every project must have an owner.
4. Every project must have a clear purpose.
5. Every project must have a status.
6. Every project must have a project record.
7. Project records must capture lessons learned.
8. Completed, cancelled, and rejected projects must remain documented.
9. Project knowledge must return to AOS.
10. No project should remain active without progress updates.

---

# 5. Project Register Principle

> AOS does not only track work.
>
> AOS tracks why work exists, what it produces, and what the institution learns from it.
