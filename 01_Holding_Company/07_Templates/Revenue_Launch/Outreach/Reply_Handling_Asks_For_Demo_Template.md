# Purpose

The reply to send when a prospect asks to see the product or an example before deciding. ALSAKKAF Systems does not have a generic public demo (each engagement is scoped around the client's own workflow) — this template is honest about that instead of improvising one.

---

# Template

```text
Hi [First Name] — fair question, and happy to be upfront about it.

Because each engagement is scoped around your own workflow rather than a one-size-fits-all product, there isn't a generic demo to send — what I can share instead is a short example of the shape of what's delivered (a real internal dashboard/report we use ourselves, not built for [Company Name] specifically), so you can see the format before we scope anything for you.

Would that be useful, alongside a short call to talk through what your version would actually look like?

Best,
[Your Name]
```

---

# How to Use

1. CEO approval is required before this message is sent.
2. Only attach or link the approved internal demonstration package (see `01_Holding_Company/04_Operations/03_Revenue_Operations/` demo materials, generated from Atlas's own operational outputs) — never a mockup presented as if it were built for the prospect.
3. Never claim a screenshot or example was built "for" the prospect unless real, delivered, paid work exists for them.
4. Log the reply and this response in the Communications Inbox Register.
